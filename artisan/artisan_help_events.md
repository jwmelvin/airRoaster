# Artisan Event Configuration Help

Copied from Artisan's help dialogs for the Event configuration (Buttons and Sliders tabs), reformatted for readability.

## Event Custom Buttons

Button numbers can be drag-and-dropped to a new position. While holding ALT (Windows) / OPTION (macOS), buttons are swapped instead.

### Button table columns

| Column | Description |
|---|---|
| Button Label | Enter `\n` to create labels with multiple lines. `\t` is substituted by the event type. |
| Event Description | Description of the event to be recorded. |
| Event Type | Event type to be recorded, or leave blank for no event. `±` types add a chosen offset (positive or negative) to the present value of the chosen event. |
| Event Value | Value of the event (1–100) to be recorded. |
| Action | Perform an action at the time of the event. |
| Documentation | The action command. Depends on the action type; `{}` is replaced by the event value (plus the offset for `±` event types). |
| Button Visibility | Hides/shows the individual button. |

### Configuration options

| Option | Description |
|---|---|
| Max buttons per row | Sets a maximum number of buttons to display on a single row. |
| Button size | Sets a size for the buttons: tiny, small, or large. |
| Color Pattern | Applies one of 99 autogenerated color patterns to the buttons. Set to `0` to choose button colors manually. |
| Mark Last Pressed | Inverts state and color of the last button pressed. |
| Tooltips | Shows the button specification as a tooltip when hovering over a button. |
| Add | Adds a new button to the bottom of the table. |
| Insert | Inserts a new button above the selected button. |
| Delete | Deletes the selected button. |
| Copy Table | Copies the button table in tab-separated format to the clipboard. Option/ALT-click to copy a tabular format instead. |
| Help | Opens this window. |

### Button commands

Notes:

- `{}` can be used as a placeholder; it is substituted by the current button value plus the offset for `±` event types. If the placeholder occurs several times in a description/command, all occurrences are replaced by the value.
- The placeholders `{ET}`, `{BT}`, `{time}`, `{ETB}`, `{BTB}`, and `{WEIGHTin}` are substituted by the current ET, BT, time, ET background, BT background value, and batch size (in g) in Serial/Artisan/CallProgram/MODBUS/S7/WebSocket commands.
- Commands can be sequenced, separated by semicolons: `<cmd1>;<cmd2>;<cmd3>`.
- All characters given as documentation to a **Serial Command** action are sent as one string to the connected device. If the device can interpret this string as separate commands separated by semicolons, this is fine. Otherwise, use a **Multiple Event** referencing several event buttons (via a comma-separated list of event button numbers as the documentation string), where each referenced button issues one command via its own Serial Command action. Those buttons can be hidden, giving the same effect as if the Serial Command allowed a sequence of commands.
- In PHIDGET commands, the optional parameter `<sn>` has the form `<hub_serial>[:<hub_port>]` and refers to a specific Phidget HUB by its serial number, optionally specifying the port the addressed module is connected to.

#### Simple actions

| Action | Command | Description |
|---|---|---|
| Serial Command | ASCII serial command or binary `a2b_uu(serial command)` | |
| Call Program | A program/script path (absolute or relative) | Starts an external program |
| Multiple Event | Button numbers or `sleep(<float>)` separated by commas: `1,2,sleep(2.5),3` | Triggers other buttons |
| p-i-d | `<p>;<i>;<d>` | Configures PID to the values `<p>;<i>;<d>` |
| Fuji Command | `write(<unitId>,<register>,<value>)` | |

## Event Custom Sliders

### Slider table columns

| Column | Description |
|---|---|
| Event | Hide or show the corresponding slider. |
| Action | Perform an action on slider release. |
| Command | Command to perform, depends on the Action type. `{}` is replaced by (slider value × Factor + Offset). |
| Offset | Offset added to the slider value (after scaling by Factor). |
| Factor | Scale factor; the slider value is multiplied by this value. |
| Min | Sets the minimum value of the slider range. |
| Max | Sets the maximum value of the slider range. |
| Course | When ticked, the slider moves in steps of 10. |
| Temp | Should be ticked when the slider's value is a temperature, so Artisan can properly scale the value between Celsius and Fahrenheit. |
| Unit | Optional text used in annotations for the units of the slider value. |

### Slider commands

Notes:

- `{}` can be used as a placeholder; it is substituted by (value × factor + offset). In all slider command actions except IO, VOUT, S7, and RC Commands, the bound value is converted from float to int.
- The placeholders `{ET}`, `{BT}`, `{time}`, `{ETB}`, `{BTB}`, and `{WEIGHTin}` are substituted by the current ET, BT, time, ET background, BT background value, and batch size (in g) in Serial/Artisan/CallProgram/MODBUS/S7/WebSocket commands.
- Commands can be sequenced, separated by semicolons: `<cmd1>;<cmd2>;<cmd3>`.
- In PHIDGET commands, the optional parameter `<sn>` has the form `<hub_serial>[:<hub_port>]` and refers to a specific Phidget HUB by its serial number, optionally specifying the port the addressed module is connected to.
- In YOCTOPUCE commands, the optional parameter `<sn>` holds either the module's serial number or its name.

#### Simple actions

| Action | Command | Description |
|---|---|---|
| Serial Command | ASCII serial command or binary `a2b_uu(serial command)` | |
| Call Program | A program/script path (absolute or relative) | Starts an external program |
| Fuji Command | `write(<unitId>,<register>,<value>)` or `sleep(<float>)` | |

#### Artisan Command

Sliders support the same Artisan commands as buttons (see [Artisan Command](#artisan-command) above), with these differences:

- No `$` variable.
- No `alarms(<bool>)`, `button(i,b)`, `button()`, or `visible(i,b)`.
- `button(<name>)` takes only a name from { START, CHARGE, DRY, FCs, FCe, SCs, SCe, DROP, COOL, OFF } — no `<bool>` form.

| `read(<json>)` | If the `<json>` text respects the JSON format, it is sent to the connected WebSocket server and the response is bound to the variable `_` |
